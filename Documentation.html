01 — Standout Features

Four high-impact AI features

Each feature goes beyond basic implementation and demonstrates meaningful ML usage. Together they form a coherent intelligent system rather than a collection of disconnected tools.

Behavioral AI

Adaptive Fear Score Engine

Moves beyond simple self-reported surveys by inferring investment anxiety from real interaction patterns — hesitation time, rejection count, scroll behavior, and decision reversals.

What makes it stand out: The score updates in real time as the user interacts, not just after a quiz. A user who hesitates 8 seconds before clicking "simulate" or who immediately scrolls to worst-case outcomes is flagged differently from one who explores best-case first. This is behavioral fingerprinting applied to financial psychology.
Probabilistic Modeling

Animated Monte Carlo Simulation

Runs 500–1000 stochastic paths of a portfolio forward in time using historical return distributions, rendering each path as an animated line on screen before settling into confidence bands.

What makes it stand out: The animation itself communicates uncertainty far better than a static chart. Users viscerally see that markets are paths, not points. The engine adapts its volatility parameters based on the user's fear score — showing a more conservative asset mix to highly fearful users by default while still exposing them to the full distribution.
NLP + Personalization

AI Portfolio Narrator

After each simulation, an LLM generates a 3–4 sentence plain-English explanation of what happened, calibrated to the user's fear level and financial literacy score derived from onboarding.

What makes it stand out: The explanation changes tone based on fear score. A high-fear user gets: "Even in the worst scenario, your investment would still be worth more than 80% of what you put in." A low-fear user gets the same data framed differently: "Your median expected return is 9.2% annually, outpacing inflation by 5 points." Same data, different emotional register — that's intelligent personalization.
Adaptive Recommendation

Dynamic Portfolio Suggester

Recommends portfolio allocations that shift as the user's fear score improves over multiple sessions, gradually introducing riskier (higher-expected-return) instruments as confidence builds.

What makes it stand out: It implements a lightweight collaborative filtering model — users who showed similar fear trajectories (fearful early, then gained confidence) and ended up investing successfully inform recommendations for current fearful users. This creates a genuine learning loop that improves with more users, not just more features.

03 — End-to-End Data Flow

From first click to continuous learning

The system processes data through six distinct phases, each with clearly defined inputs, transformations, and outputs. Every phase produces data that feeds downstream phases and eventually loops back into model retraining.

01

User input capture

The React frontend captures three categories of data simultaneously: explicit inputs (onboarding quiz answers, investment amount slider, time horizon), behavioral signals (hover durations recorded via onMouseEnter/Leave, click timestamps, scroll position snapshots every 2s), and session metadata (device type, session start time, return visit flag). All behavioral events are batched and sent to the backend every 15 seconds via a lightweight POST /behavior/batch endpoint to avoid network overhead.

02

Fear score computation

On the first page load, a pre-trained Random Forest model runs synchronously on the server using the quiz answers and any prior session data for returning users. For new users, the default class is "medium" with 50% confidence. As behavioral events accumulate within the session, the score is re-computed in a background task every 60 seconds and pushed back to the frontend via a WebSocket or simple long-poll. The score is stored per-session in behavior_sessions and rolled into the user profile after session end.

03

Monte Carlo simulation execution

When the user clicks "Run simulation," the frontend sends a POST /simulation/run request containing the portfolio allocation percentages, investment amount, and time horizon. The backend retrieves pre-computed mean return and standard deviation values for each asset class from a Redis cache (updated nightly from yfinance) and runs 1000 paths using NumPy's vectorized random normal sampler. The P5, P25, P50, P75, and P95 quantiles are extracted and returned as a JSON array of ~50 time points per quantile — roughly 2KB of data, rendering in under 200ms.

04

AI narration generation

The simulation result object (P5/P50/P95 values, asset allocation, time horizon) is combined with the user's fear score class and literacy score and injected into a carefully structured prompt sent to the Claude or OpenAI API. The prompt instructs the model to generate exactly 3 sentences: one anchoring the median outcome, one acknowledging the downside risk honestly, and one forward-looking. The response is streamed token-by-token to the frontend so it appears to type out in real time, which adds perceived intelligence and keeps users engaged.

05

Portfolio recommendation

After narration, the recommendation engine runs. It queries the user_profiles table for users with matching fear class and demographic bracket, retrieves their accepted portfolio suggestions, and uses a cosine similarity ranking to find the top 3 allocations most accepted by similar users. These are returned with confidence scores and a brief rule-based explanation of why each is suggested (e.g., "80% of users with your profile chose this after 2 sessions"). For the hackathon demo, this can be a lookup table seeded with realistic synthetic data.

06

Feedback collection and model retraining loop

Every user interaction with recommendations (accepted/dismissed), simulation replays (did they reduce the amount or increase it?), and session return rate feeds back into labeled training data. A nightly job (retrain.py) reads the last 7 days of labeled behavior from the database, retrains both the Fear Score classifier and the literacy estimator using scikit-learn, serializes the new models with joblib, and replaces the loaded models in the FastAPI application state. This is a complete, demonstrable continuous learning loop.

04 — UI/UX Flow

Seven screens, one coherent journey

Each screen has a single primary job. The navigation is intentionally linear for new users and non-linear for returning ones — first-timers are guided through a fixed flow while experienced users can jump directly to simulation or portfolio views.

Screen 1 — Onboarding quiz (first visit only)

A 3-question card-based quiz that takes under 60 seconds. Questions are designed to feel like a personality quiz, not a financial form, to reduce initial anxiety. Each question is displayed one at a time with a smooth slide transition.

Question 1: "If your investment dropped 15% in a month, what would you do?" → Sell immediately / Wait and see / Buy more. Question 2: "What's your goal?" → Short-term gains / Long-term growth / Just learning. Question 3: "How much are you thinking of starting with?" → Range slider from ₹500 to ₹1,00,000.

After submission, an animated "calculating your profile..." state plays for 2 seconds (even if the result is instant) to make the AI feel substantive before the dashboard loads.

Card slider (1 card at a time) Multiple choice buttons Range slider Progress indicator (3 dots) 2s AI calculation animation

Screen 2 — Main dashboard

The primary home screen showing the user's current fear score as an animated radial gauge (0–100), their financial literacy score as a progress bar, a "next steps" prompt card, and a row of recent simulation results if they exist. The fear score gauge uses color encoding: red at high fear, amber at medium, green at low — shifting in real time as sessions accumulate.

The key design principle is that the dashboard should feel like progress, not assessment. The headline copy changes based on fear level: high fear sees "Let's explore safely, no real money involved." Low fear sees "You're building real confidence. Ready for the next scenario?"

Animated fear score gauge Literacy score bar Personalized headline copy Recent simulations row CTA button to simulation Educational tip card (rotates daily)

Screen 3 — Simulation interface

The centrepiece of the product. A split-layout screen with controls on the left and the live simulation canvas on the right. Controls include: investment amount (slider + number input), time horizon (5 / 10 / 20 years), asset allocation sliders for 4 asset classes that must sum to 100% (validated live), and a "risk preset" row with Conservative / Balanced / Growth buttons that snap the allocation sliders.

When the user clicks "Run simulation," the chart canvas triggers the animated Monte Carlo playback: 1000 thin grey paths draw simultaneously over 1.5 seconds, then fade to reveal the P5/P50/P95 bands in color. The animation stops at the user-selected time horizon. This animation is the single most impressive visual moment in the demo — it must run smoothly.

Investment amount slider Time horizon selector 4× asset allocation sliders (sum to 100%) Risk preset buttons Animated Monte Carlo canvas (Recharts/D3) P5/P50/P95 band overlay Loss Probability Meter (badge)

Screen 4 — Simulation results

Appears below the simulation canvas after it completes. Contains three stat cards (Best case / Expected / Worst case) showing the final portfolio value in large numbers, a "Loss probability" badge (e.g., "32% chance of loss"), and the AI narrator block where the generated explanation streams in.

Below the narration, a "What does this mean?" expandable section uses rule-based text to explain key terms (volatility, confidence interval, compound growth) in plain language. This is intentionally not AI-generated — it's static, reliable copy that won't hallucinate financial terms.

3× outcome stat cards (best/expected/worst) Loss probability badge (color-coded) AI narration streaming block Expandable "what does this mean?" glossary Save simulation button Re-run with different settings CTA

Screen 5 — Behavioral insights

Shows the user their own behavioral data over time — how their fear score has changed across sessions, which scenarios they explored most, their average decision time per simulation, and a "hesitation heatmap" showing which part of the UI they spend the most time before acting. This turns behavioral data collection from surveillance into a feature the user actually values.

The fear score timeline chart is the most compelling element here — users can literally watch their anxiety decrease session by session. The copy reads: "You've been getting braver. Your fear score dropped 28 points since your first session."

Fear score timeline chart Session count and streak Decision time trend Most-explored scenario tags Personalized insight copy

Screen 6 — Portfolio suggestions

Shows 3 recommended portfolio allocations ranked by ML similarity score. Each recommendation is presented as a card with an allocation pie chart, expected annual return range, volatility label, and a "why this for you" sentence generated from the recommendation engine's reasoning. Users can click any recommendation to pre-load it into the simulation interface instantly.

A "compare" toggle lets users place two recommendations side by side against the same simulation parameters, making the risk-return tradeoff visually immediate. The "confidence meter" beneath each card shows how many similar users adopted this portfolio successfully.

3× portfolio recommendation cards Mini allocation pie charts Expected return range badge "Why this for you" AI text Side-by-side compare toggle "Load into simulator" CTA

Screen 7 — Investment confidence card

A shareable "achievement" card generated when the user's fear score drops below a threshold (below 40 for the demo). It shows their starting fear score, current fear score, total simulations run, most-explored portfolio type, and an AI-generated personal statement: "You've simulated ₹50,000 invested over 10 years. You've seen the worst case, understood the odds, and kept going. That's what an investor looks like."

The card has a "Copy image" and "Share" button, turning it into organic marketing. Judges will love this: it's the product's emotional payoff made tangible and shareable.

07 — Implementation Plan

Four days to a demo-ready MVP

This plan is structured for a team of 3–4 people working in parallel. Day 1 produces a skeleton everyone can plug into. Day 4 is reserved entirely for polish and the demo script — a rough product presented confidently beats a perfect product demoed nervously.

Day 1
Foundation
  • Set up React + Vite + Tailwind frontend skeleton with all 7 route stubs
  • Set up FastAPI backend with all API endpoints returning mock JSON
  • Set up PostgreSQL schema (run migrations), connect SQLAlchemy
  • Agree on API contracts (request/response shapes) in a shared doc
  • Implement onboarding quiz UI end-to-end with real POST to backend
  • Deploy frontend to Vercel, backend to Railway (do this early!)
Day 2
Core simulation
  • Implement Monte Carlo engine in NumPy (vectorized, 1000 paths)
  • Hardcode asset_parameters table with realistic σ/μ for 4 asset classes
  • Build simulation interface UI: sliders, preset buttons, allocation controls
  • Implement animated simulation canvas using Recharts LineChart with custom renderer
  • Wire simulation controls → POST /simulation/run → render confidence bands
  • Build results section: 3 stat cards + loss probability badge
Day 3
AI + ML
  • Train Fear Score classifier on synthetic dataset (100 labeled rows)
  • Integrate fear score: POST onboarding → run classifier → store in DB
  • Implement behavior event logging from frontend (batch POST every 15s)
  • Build AI narration: construct prompt, call API, stream response to UI
  • Seed portfolio recommendations table with 10 pre-made allocations
  • Build portfolio suggestions screen with recommendation cards
  • Build insights screen with fear score timeline chart
Day 4
Polish + demo
  • Seed 3 demo user personas with pre-built behavior history (conservative / balanced / growth)
  • Build confidence card screen + canvas export to PNG
  • Add loading skeletons, error states, and empty state copy everywhere
  • Record 5-minute demo video as backup plan
  • Write demo script: onboard → simulate → read narration → see improvement → share card
  • Practice the demo 3 times. Time it. Cut anything that slows it down.

Smart simplifications that save time without sacrificing demo quality:

SIM-01

Hardcode asset parameters

Use pre-computed historical mean/std for 4 asset classes rather than live API calls. Nifty 50 Index: μ=12%, σ=18%. Government bonds: μ=7%, σ=4%. Gold: μ=9%, σ=14%. Crypto (optional): μ=45%, σ=80%. These are realistic and defensible.

ML-01

Synthetic training data

Generate 200 synthetic behavior records programmatically in Python using numpy.random, assign labels manually using a scoring rule (decision_time > 5s AND rejection_count > 2 → "high"), then train the classifier. It will generalize fine for a demo.

REC-01

Seed recommendations table

Instead of computing collaborative filtering live, pre-populate the portfolio_recommendations table with 9 well-designed allocations (3 per fear class) and return the matching ones at query time. Label them as "chosen by 78% of users like you" for credibility.

DB-01

Use Supabase free tier

Supabase gives you a hosted PostgreSQL instance with a REST API, auth, and a dashboard for free. It eliminates database setup entirely — just run the schema SQL in their console and use their Python client. Save 3–4 hours of infrastructure work.

08 — Hackathon Differentiation

How to win the room

Most teams will build a simulation tool. You're building a story about transformation. Every design choice should reinforce that narrative — a user who arrives fearful and leaves confident. Here's how to make that unmistakable.

DEMO-01

Show the fear score dropping live

Pre-seed a "demo" user account with 3 prior sessions showing a fear score of 82. During the demo, run 2 simulations and show the fear score drop from 82 to 61 in real time. Judges will see the product actually learning — that's the moment that wins.

DEMO-02

Animate 1000 paths for 1.5 seconds

The Monte Carlo animation is your visual payoff. Each of 1000 grey paths should draw simultaneously over 1.5 seconds, then smoothly fade to the colored confidence bands. This single animation communicates statistical uncertainty better than any slide ever could. Practice running it on the projector before the demo.

DEMO-03

Stream the AI narration live

Don't show a preloaded response — actually call the API during the demo and stream it token-by-token. It types out in front of the judges. The moment where an AI produces a sentence calibrated to the user's fear level, in plain language, in real time, is unforgettable.

DEMO-04

End with the confidence card

Close the demo by triggering the confidence card generation for the demo user. Show it on screen, then actually export it to PNG. A product that ends with a shareable artifact feels complete and polished in a way that pure data tools do not. Judges remember endings.

PITCH-01

Lead with the problem, not the tech

Open with: "70% of young Indians want to invest but don't, because they're afraid of losing money. They have nowhere safe to practice. We built that safe place." Only then show the product. Judges fund solutions to real problems, not impressive ML models.

PITCH-02

Prepare 3 pre-built personas for judges

Have three demo accounts ready: Priya (high fear, first timer), Arjun (medium fear, 3 sessions in), and Kavya (low fear, ready to invest). Each shows a different stage of the journey. Judges can "be" any user and see what the product does for that person specifically.
